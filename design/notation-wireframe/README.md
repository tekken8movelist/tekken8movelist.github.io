## Part 1 · 按键图组件（hifi）

### 设计决策（用户已确认）
- 布局：**街机 2×2 方阵**（非手柄 45° 菱形）。1 左拳=左上、2 右拳=右上、3 左脚=左下、4 右脚=右下。
- **单色**，不用 T7 彩色编码。4 键**常显**：未按 = 幽灵态（7% 填充 + 32% 描边，**不显数字**），按下 = `.on` 实心 + 反色数字 + 微辉光（浅色主题自动关辉光）——只有按下的键有数字，一眼可读。
- 方向 = **粗箭头**（clip-path 绘制，非字符字形），8 方向靠 transform 旋转；**按住**（大写 F/B/U/D）= 箭头套圆圈；N = 文字。
- 状态前缀（起身中/凤凰中/背身时/蹲伏中/横移中）保持**文字胶囊** `.tk-state`，不做图标。
- 角色主题色 `--tk-acc` **只用于状态胶囊/标签**，按键图与箭头恒为单色。默认中性灰蓝，勿写死粉色。

### 标记约定
每条指令一个 `.tk-in`，内部按 token 顺序排列：

```html
<link rel="stylesheet" href="tekken-input-notation.css">

<!-- d/f+2, 3（月燕） -->
<span class="tk-in tk-md">
  <span class="tk-dir df"></span><span class="tk-sep">+</span>
  <span class="tk-b"><i>1</i><i class="on">2</i><i>3</i><i>4</i></span>
  <span class="tk-sep">›</span>
  <span class="tk-b"><i>1</i><i>2</i><i class="on">3</i><i>4</i></span>
</span>
```

- `.tk-b` 恒有 4 个 `<i>`，顺序固定 1,2,3,4；按下加 `.on`；同按（1+2）= 同一方阵多个 `.on`
- `.tk-dir` 方向 class：`f b u d df db uf ub`；按住加 `.hold`
- `.tk-sep` 分隔符：`›` 接续（逗号）、`+` 方向+键、`~` 滑键、`＊` 蓄力
- `.tk-state` 状态前缀胶囊；`.tk-n` 回中 N
- 尺寸：`.tk-sm` 11px（列表行，配 `.tk-b.no-num` 隐藏数字）/ `.tk-md` 15px / `.tk-lg` 22px；组件全 em 计量，改 font-size 即缩放

### 解析映射（td.cmd 文本 → 组件标记）
写一个 `renderCmd(text)`：
- 前缀词（起身中|凤凰中|背身时|蹲伏中|横移中|热能中|催眠中|愤怒中…）→ `.tk-state`
- 方向 token（f/b/u/d/df/db/uf/ub，含 `d/f` 写法；大写 = `.hold`）→ `.tk-dir`
- 数字与 `+` 组合（`1+2`）→ 一个 `.tk-b`，对应 `<i>` 加 `.on`
- `,` → `<span class="tk-sep">›</span>`；`~` → 滑键分隔；`*` → 蓄力 `＊`
- 无法解析的片段（如"看准对手拳击"）原样输出文字，别硬转

### Design Tokens
- `--tk-ink` 前景（默认 `#dce3ea`）/ `--tk-bg` 底色（默认 `#0e1116`）；`.light` = `#1d1d1f` / `#ffffff`
- `--tk-acc` 角色主题色，默认 `#8b98a5`；**44 名角色（至第3季）推荐色**见 CSS 底部 `.tk-char-*` token（基础 32 + S1：艾迪/莉迪亚/平八/克莱夫 + S2：安娜/法昆拉姆/铠甲王/米亚莉·佐 + S3：州光/鲍勃/罗杰Jr./范马勇次郎）
- 关键比例（em，基于 .tk-in 字号）：键格 1.35em、格间距 .2em、圆角 .34em、描边 .09em；箭头容器 1.5em、箭身 1.02em、clip-path `polygon(10% 18%, 94% 50%, 10% 82%, 32% 50%)`
- 字体：`ui-monospace, "JetBrains Mono", Consolas, monospace`

## Part 2 · 移动端出招表（lofi 结构参考）

### 屏幕 1：出招表列表
- 顶栏：角色名 + 记法切换 seg（文字 df+2 / 图标按键图，全局记忆，localStorage）
- 搜索框（招式名 / 指令）+ 分类 chips（全部/投技/打击/凤凰/背身/催眠）
- 分组列表，粘性分组头（打击技 ATTACKS…）；右缘竖排分组快速跳转
- 行内容：招式名 + 标签（转背身/HEAT/架势）、指令（sm 按键图或文字）、判定色点（上=蓝 中=橙 下=红 特=紫，沿用现有 --hi/--mid/--low/--misc）、伤害
- 行内可选"详模式"：直接显示 发生/被防/命中 三列（被防可确反标红、命中有利标绿）

### 屏幕 2：招式详情（底部弹层）
- 半屏：招式名 + 指令（md 按键图）+ 判定 chips + 帧数四格（发生/被防/命中/伤害，大数字）
- 上滑全屏：动作预览占位（视频/GIF）、要点文案（来自现有攻略页 KEY MOVES）、相关连招卡、「＋加入练习清单」主按钮、上一招/下一招翻页
- 弹层关闭回列表，滚动位置保留

### 屏幕 3：练习清单（可后做）
- 每招/每连招可从详情"＋"加入；条目 = 招式 + 按键图 + 目标次数 + 勾选
- 顶部进度条 + 连续天数

### Interactions
- 记法切换即时生效于全部指令渲染，无刷新
- 弹层手势：上滑扩展、下滑关闭
- 判定/帧数筛选（后续）：chips 组合过滤 + 结果计数
